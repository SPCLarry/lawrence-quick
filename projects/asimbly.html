PROJECT CASE STUDY

Asimbly

A deep dive into the system architecture of a collaborative FPS project. Focusing on scalable C++ foundations and Blueprint implementation.

System Breakdowns

1. Character Controller 2. UI Architecture 3. Material Design 4. AI Behavior 5. Interaction Logic

1. Character Systems

Goal: Create a responsive "True First Person" character supporting Sliding and Mantling.

I created logic specific actor components to handle custom movement behaviors like sliding and mantling.

Above: Loading...

2. UI Architecture

Goal: Decouple UI Widgets from data storage to ensure settings persist across levels.

I implemented a custom C++ GameInstance Subsystem. The UI widgets act strictly as "Views," querying the Subsystem ("Model") for data. This MVVM-style approach increases maintainability and reduces coupling.

Above: Loading...

3. Material Design & HLSL

Goal: Create performant, dynamic post-processing effects that enhance visual feedback without impacting performance.

I created HLSL shaders to create glitch and damage feedback effects that are lightweight and performant.

Above: Loading...

4. AI Design

Goal: Tactical enemy behavior using Behavior Trees and EQS.

The AI utilizes a Behavior Tree with a custom Service for Line-of-Sight checking. I used the Environment Query System (EQS) to allow the AI to find optimal range positions relative to the player.

Above: Loading...

5. Interaction Systems

Goal: A complex world based and UI interaction system that feels intuitive and responsive.

I created a BPI_Interact interface and a Interaction State Machine. The player can either interact traditionally by looking at objects or via a free-look cursor for more precise control.
